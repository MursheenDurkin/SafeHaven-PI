# Contributing

SafeHaven Pi is an open source project and contributions are very welcome.

If you've improved something, fixed a bug, added a feature, or just have an idea — please share it.

---

## How to contribute

1. Fork the repository on GitHub
2. Create a branch for your changes: `git checkout -b feature/your-idea`
3. Make your changes and test them on a Raspberry Pi
4. Commit with a clear message: `git commit -m "add: description of what you did"`
5. Push your branch: `git push origin feature/your-idea`
6. Open a Pull Request and describe what you changed and why

---

## What we'd love to see

- Support for additional VPN providers
- Improved captive portal handling for hotels and airports
- Dashboard improvements
- Support for Raspberry Pi 4 or other hardware
- Additional language support for the menu
- Better documentation or setup guides
- Security improvements or hardening suggestions

---

## Guidelines

- Keep it simple — this project is designed to be accessible to non-technical users
- Test on real hardware where possible
- Don't commit real credentials, passwords, or private keys
- Plain English comments in scripts are encouraged
- If you're adding a new feature, update the README to reflect it

---

## Questions or ideas?

Open an issue on GitHub and let's talk about it. All ideas are welcome, no matter how small.
